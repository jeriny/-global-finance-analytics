# Global Finance Analytics

> 글로벌 부서의 손익, KPI, 경쟁사 데이터를 통합 관리하기 위해 구축한 **재무 분석 대시보드 시스템**입니다.
> 기존 엑셀·수작업 중심의 데이터 관리 흐름을 웹 기반 조회·입력·권한 관리 구조로 개선한 프로젝트입니다.

---

## 1. 프로젝트 소개

Global Finance Analytics는 글로벌 매출, 손익, KPI, 경쟁사 데이터를 웹 대시보드에서 통합적으로 조회·분석할 수 있도록 만든 내부 업무 자동화 시스템입니다.

초기에는 빠른 검증을 위해 Streamlit 기반 프로토타입으로 시작했으며, 이후 실제 운영 환경을 고려해 **React SPA + FastAPI + PostgreSQL** 기반의 웹 서비스 구조로 확장했습니다.

## 2. 프로젝트 배경

기존 업무에서는 손익, KPI, 경쟁사 관련 데이터가 여러 엑셀 파일과 수작업 프로세스에 분산되어 있었습니다.

이로 인해 다음과 같은 문제가 있었습니다.

* 데이터 조회와 공유 과정이 번거로움
* 파일 버전 관리와 수동 집계 부담 발생
* 사용자별 조회·수정 권한 구분이 어려움
* 데이터 변경 이력과 다운로드 이력 추적이 제한적
* 보고용 자료 제작 과정에서 반복 작업 발생

이를 해결하기 위해 데이터를 웹 서비스에서 통합 조회하고, 권한별로 입력·수정·관리할 수 있는 구조를 설계했습니다.

## 7. 기술적 고민과 해결

### 1. 프로토타입에서 운영 가능한 서비스 구조로 전환

초기에는 빠른 검증을 위해 Streamlit을 사용했지만, 실제 업무 환경에서는 사용자 권한, 관리자 기능, 대용량 테이블, 리포트 다운로드, 배포 환경이 필요했습니다.

이를 위해 프론트엔드와 백엔드를 분리한 React SPA + FastAPI 구조로 전환하고, 데이터는 PostgreSQL에서 관리하도록 재설계했습니다.

### 2. 사용자 권한과 데이터 수정 범위 분리

재무 데이터를 다루는 시스템이기 때문에 모든 사용자가 동일한 권한을 갖는 구조는 적절하지 않았습니다.

사용자 역할을 조회자, 편집자, 관리자 수준으로 나누고, 화면 접근과 API 요청 권한을 분리하여 데이터 수정 범위를 통제했습니다.

### 3. 엑셀 기반 업무 흐름의 웹 서비스화

기존 데이터는 엑셀 중심으로 관리되었기 때문에, 업로드된 원본 데이터와 대시보드에서 사용하는 정제 데이터를 분리할 필요가 있었습니다.

이를 위해 원본 적재 영역과 서비스 조회 영역을 구분하고, 데이터 반영 이후 관련 캐시를 무효화하여 최신 데이터가 화면에 반영되도록 설계했습니다.

### 4. 보고용 PDF/Excel 다운로드 구현

대시보드에서 조회한 내용을 보고 자료로 활용할 수 있도록 PDF 및 Excel 다운로드 기능을 구현했습니다.

차트와 테이블이 포함된 화면을 여러 페이지로 나누어 출력하고, 다운로드 이력을 감사 로그에 남겨 내부 관리 요구사항을 충족하도록 했습니다.

### 5. JWT + Refresh Token + CSRF 기반 인증 구조 구현

재무 데이터를 다루는 내부 시스템이기 때문에 단순 로그인 기능뿐 아니라, 세션 관리와 상태 변경 요청에 대한 보안 처리가 필요했습니다.

이를 위해 Access Token은 Bearer JWT 방식으로 사용하고, Refresh Token은 HttpOnly Cookie로 관리하는 구조를 적용했습니다. Refresh Token은 재발급 시 회전하도록 설계했으며, 재사용이 감지될 경우 기존 세션을 무효화해 토큰 탈취 상황에 대응할 수 있도록 했습니다.

또한 데이터 수정, 권한 변경, 관리자 기능처럼 상태를 변경하는 요청에는 CSRF 방어 정책을 적용했습니다. 프론트엔드 요청에는 인증 토큰과 요청 헤더를 함께 사용하도록 구성하고, 주요 이벤트는 감사 로그에 기록해 보안성과 추적 가능성을 높였습니다.

## 8. 성과

* 기존에 엑셀과 수작업 중심으로 관리되던 글로벌 매출·손익·KPI 데이터를 **웹 기반 대시보드에서 통합 조회**할 수 있도록 전환했습니다.
* KPI 요약, 손익 상세, 품목별 매출, 연결 손익 등 주요 재무 화면을 구성하여 **분산된 재무 데이터를 하나의 서비스 흐름으로 정리**했습니다.
* 사용자 역할을 조회자, 편집자, 관리자 수준으로 분리하여 **권한별 조회·수정 범위를 통제**할 수 있는 구조를 구현했습니다.
* 엑셀 업로드, 비용 배분 입력, 리포트 다운로드, 관리자 승인 등 실무 운영에 필요한 기능을 포함해 **단순 대시보드가 아닌 내부 업무 시스템 형태**로 확장했습니다.
* 로그인, 권한 변경, 다운로드 등 주요 이벤트를 감사 로그로 기록하여 **사용자 활동 추적과 운영 관리 기반**을 마련했습니다.
* 경쟁사 재무 정보, 뉴스 데이터, AI 요약 기능을 연동하여 **내부 재무 데이터와 외부 참고 데이터를 함께 확인**할 수 있도록 했습니다.
* Docker 기반 실행 환경과 README/인수인계 문서를 정리하여 **후속 담당자가 프로젝트 구조와 실행 흐름을 이해할 수 있는 기반**을 마련했습니다.
